# 更新日志

本文件用于记录每个正式版本的更新内容。GitHub Actions 发布 `v*` 标签时，会读取对应版本小节并写入 GitHub Release 和 `latest.json.notes`，应用内“检查更新”弹窗也会展示这里的内容。

## [2.0.1] - 2026-04-29

### 新增

- 项目升级为打包部署工作台

### 优化

- 部署中心服务器列表从 List 布局重构为 Table 布局，支持分页浏览，信息展示更紧凑清晰。
- 部署中心服务器管理新增搜索功能，支持按名称、地址、分组关键词快速筛选服务器。
- 服务页面服务器列表新增搜索功能，支持按名称、地址关键词筛选，并增加分页控制。
- 部署、产物、服务三个页面自动隐藏侧边栏，主工作区获得更大展示空间。
- 支持快捷切换项目

### ⚠️ 注意

- 本次更新服务映射数据结构存在不兼容变更，历史服务映射配置可能无法正常加载，升级前请备份 `app.sqlite3` 数据库文件，慎重更新。

### 移除

- 移除部署、产物、服务页面的侧边栏冗余信息面板（部署上下文、产物筛选、服务器资源与最近部署），减少信息噪音。

## [2.0.0] - 2026-04-29

### 新增

- 新增 Maven 模块依赖图分析，支持查看模块依赖、被依赖、父子聚合关系、循环依赖提示与推荐联动模块。
- 新增"部署中心"，包含服务器管理、服务映射配置和一键部署入口。
- 新增服务器密码安全存储，密码在本地持久化前使用 Windows DPAPI 加密，不再以明文形式散落在配置中。
- 新增本地部署闭环流程，支持基于构建产物执行上传、停止旧服务、替换文件、启动服务、健康检查并记录部署历史。
- 新增可编排部署流水线，支持 6 种步骤类型：SSH 命令、等待、端口检测、HTTP 健康检查、日志关键字检测、文件上传，每个步骤可独立配置超时、重试次数和失败策略（停止/继续/回滚）。
- 新增 3 套内置部署模板：Spring Boot Jar 滚动替换、Tomcat War 替换、静态站点压缩包发布，支持基于模板快速创建服务映射。
- 新增启动探针检测，部署完成后自动验证服务是否真正启动，支持进程探针（PID 文件）、端口探针、HTTP 探针和日志探针四维检测，可配置超时时间、检测间隔和成功策略。
- 新增部署备份与回滚机制，部署前自动备份旧版本，支持配置备份保留数量、备份目录、失败自动回滚和回滚后重启旧版本。
- 新增部署文件上传进度实时展示，SFTP 优先传输，SFTP 不可用时自动降级为 Base64 分块上传，上传速度和进度实时反馈。
- 新增 SSH 原生连接库替代系统 ssh/scp/plink/pscp，密码认证和私钥认证均通过 Rust SSH 库直连，不再依赖本机安装 SSH 客户端。
- 新增部署停止功能，支持随时中断正在执行的部署任务，当前步骤完成后立即终止后续步骤。
- 新增"产物"页面，集中展示当前和历史构建产物，支持定位文件和删除。
- 新增"服务"页面，展示所有服务映射的部署状态、启动探针结果和远端进程信息。
- 新增右侧检查器抽屉（Inspector Drawer），统一承载构建日志、部署日志、诊断结果和部署详情，支持构建/部署日志源切换。
- 新增底部操作栏（Bottom ActionBar），构建状态、目标模块、命令编辑和一键构建/停止操作始终可见。
- 新增构建后续操作面板，构建成功后可直接选择服务映射和服务器发起部署，实现"构建→部署"一键衔接。
- 新增部署拓扑服务，根据模块路径和产物匹配规则自动关联构建产物与服务映射。
- 新增导航状态管理，主界面改为"构建/产物/部署/服务/环境/历史"六页签 ActivityBar 导航，运行中的构建和部署显示动态徽标。
- 新增上传进度和部署日志独立状态管理，部署过程实时更新不干扰其他交互。

### 优化

- 主工作台升级为 ActivityBar + Sidebar + MainWorkspace + Inspector 四区布局，功能导航更清晰。
- 部署中心面板重构为服务器管理、服务映射配置、部署执行和部署历史四合一视图，配置与执行一体化。
- 部署历史表增强，展示部署阶段流水线进度、每步耗时、探针检测结果和回滚状态。
- 停止旧服务逻辑增强，自动识别端口占用并精准查杀 Java 进程，避免误杀非 Java 服务；支持 lsof/ss/fuser/proc 多种端口探测方式兼容不同 Linux 发行版。
- 部署命令模板支持丰富的变量替换（${artifactPath}、${remoteDeployPath}、${pidFile}、${logFile}、${portProbePort} 等），减少硬编码。
- 旧版自定义命令自动迁移为流水线步骤格式，兼容历史部署配置。

### 移除

- 移除任务流水线（Task Pipeline）功能及相关代码（TaskPipelinePanel、TaskPipelineHistoryTable、task_pipeline_executor、pipeline_repo），由部署流水线统一替代。
- 移除独立模板面板（TemplatePanel），构建模板功能合并到构建配置区。
- 移除对系统 ssh/scp/plink/pscp 的依赖，部署传输改为 Rust SSH 库直连。

### 说明

- 当前远端部署默认按 Linux/类 Unix 目录与命令习惯执行 `mkdir/mv` 等操作，适合开发/测试环境闭环。
- SFTP 传输优先，若服务器不支持 SFTP 子系统会自动降级为 Base64 分块上传，无需额外配置。

## [1.3.2] - 2026-04-23

### 新增

- 构建进程使用 Windows Job Object 管理，停止构建时可通过 Job Object 终止整个进程树，确保 Maven 及其子进程（如 javac、surefire 等）被彻底清理。
- 取消构建新增"停止中"过渡状态，停止按钮在停止请求处理期间自动禁用，避免重复点击。
- 取消构建改为异步执行，停止操作在后台线程完成进程终止与状态确认，不再阻塞主界面。
- 取消构建全流程日志反馈，从发送停止请求到进程退出，每一步操作均在日志中输出系统提示。
- 模块树加载时展示 Spin 加载动画和"正在解析项目模块..."提示，替代空白等待。

### 优化

- 构建摘要区域从 Card 布局重构为 Dock 布局，状态标签、目标模块和操作按钮在同一行紧凑排列，减少纵向空间占用。
- 构建产物列表改为紧凑模式，最大高度 120px 内滚动展示，不再撑开构建摘要区域。
- 命令预览区域移至构建配置 Tabs 下方，操作命令与配置参数更紧密关联。
- 停止构建采用 Job Object 终止 + PowerShell 进程树清理 + taskkill 三重策略，提升进程清理可靠性。
- 切换项目时自动重置模块选择、构建状态和诊断结果，避免残留数据干扰新项目。

### 修复

- 修复停止构建后 Maven 子进程（如 javac、surefire）可能继续运行未被终止的问题。
- 修复取消构建时界面短暂无响应的问题，停止操作现在在后台线程执行。
- 修复构建进程启动与停止请求并发时可能导致状态不一致的问题。

## [1.3.1] - 2026-04-23

### 新增

- 环境中心新增“环境方案”能力，支持保存、切换和删除多套 JDK、Maven、settings.xml、本地仓库与 Maven Wrapper 配置。
- 支持“自动识别 / 指定环境方案”两种环境模式，切换方案后会自动刷新环境检测结果并重新生成构建命令。

### 优化

- 重构环境设置数据结构，将环境覆盖项统一收敛到环境方案中，减少配置状态分散带来的维护成本。
- 优化环境中心窄栏布局，方案选择、方案保存、路径覆盖和执行器切换在中间栏宽度下更紧凑稳定。
- 手动覆盖路径区改为标签化布局，路径输入框自适应宽度，操作按钮统一为图标按钮，减少横向挤压。

### 修复

- 修复本地仓库路径没有正确读取 `settings.xml` 中 `<localRepository>` 的问题。
- 修复 `<localRepository>` 使用命名空间、换行、注释或 `${user.home}`、`${env.xxx}` 等变量时无法识别的问题。
- 修复环境中心路径过长导致横向滚动、按钮被挤出可视区域的问题。

## [1.3.0] - 2026-04-22

### 新增

- 将原“环境检测”升级为“环境中心”，统一展示 JDK、Maven、Maven Wrapper、settings.xml、本地仓库和 Git 状态。
- 环境中心支持一键刷新检测，并展示环境来源，包括自动识别、手动覆盖、Wrapper 和未识别状态。
- 支持手动覆盖 JDK、Maven、settings.xml 和本地仓库路径，覆盖后会立即刷新环境并重新生成构建命令。
- 手动指定 `settings.xml` 和本地仓库后，构建命令会自动追加 `-s` 与 `-Dmaven.repo.local` 参数。
- 构建失败后自动生成规则诊断结果，支持识别 JDK 版本不匹配、Maven 缺失、Wrapper 失效、settings.xml 缺失、依赖下载失败、私服不可达、profile 不存在、模块路径错误、单元测试失败和未知错误。
- 右侧日志区域新增诊断面板，展示错误摘要、可能原因、建议动作和高价值关键日志行，并支持一键复制诊断结果。

### 优化

- 环境中心卡片改为更稳定的单列排布，避免路径、版本号和状态标签在中间栏宽度下挤压换行。
- 执行器切换从长条开关改为 `Maven / mvnw` 分段按钮，Wrapper 不存在时仅禁用 `mvnw` 选项。
- 手动覆盖路径输入区改为自适应布局，避免按钮文本较长时挤压输入框。
- 日志解析和诊断规则拆分为独立前端服务，为后续接入 AI 分析预留扩展点。

### 修复

- 修复 React 开发模式下构建日志事件监听可能重复注册，导致实时日志每行显示两次的问题。
- 修复构建日志接收到相邻重复事件时没有去重的问题。
- 修复环境中心部分状态标签和路径文本显示不全、换行错乱的问题。
